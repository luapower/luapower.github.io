A | requires: ffilightweight ffi binding of the [cairo graphics] library with the following features:

| requires: ffi * cairo types have associated methods, so you can use context:paint() instead of cairo.cairo_paint(context) | requires: ffi * pointers to objects for which cairo holds no references are bound to Lua's garbage collector to prevent leaks | requires: ffi * ref-counted objects have a free() method that checks ref. count and a destroy() method that doesn't. | requires: ffi * functions that work with char* are made to accept/return Lua strings. | requires: ffi * output buffers are optional - if not passed on as arguments, temporary buffers are allocated instead; the values in the buffers are then returned as multiple return values, such as in context:clip_extents([dx1][,dy1][,dx2[,dy2]) -> x1, y1, x2, y2, where dx1 etc. are double[1] buffers. | requires: ffi * the included binary is built with support for in-memory surfaces, recording surfaces, ps surfaces, pdf surfaces, svg surfaces, win32 surfaces, win32 fonts and freetype fonts.

See | requires: ffithe [cairo manual] for the function list, remembering that method call style is available for them.

Additional | requires: ffiwrappers are provided for completeness:
